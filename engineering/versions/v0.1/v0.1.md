# v0.1 — Sign in, register, find the ready version

*Last updated: 2026-06-20*

**Status:** ✅ Complete · **Started:** 2026-06-12 · **Completed:** 2026-06-20

> Sign in with GitHub, register a product to its repository, and confirm it has a ready published
> image to deploy. Drydock ships pre-built images — building them is CI's job. Verified end to end:
> the full arc resolved `Ready` live against a real release (secrets-vault `v0.1.0`).

### Iteration 1 — Sign in: gate Drydock behind GitHub sign-in

- [x] Ability to sign in with GitHub to reach the dashboard.
- [x] Ability to restrict the dashboard to an allowlist.
- [x] Ability to stay signed in across requests.

### Iteration 2 — Register products: manage the products Drydock can deploy

- [x] Ability to create, list, view, update, and remove a product.
- [x] Ability to select the repository that defines a product.

### Iteration 3 — Find the ready version: confirm a product has a ready image to ship

- [x] Ability to resolve a product's latest released version from its repository.
- [x] Ability to see whether that version has a published image ready to deploy.

### Iteration 4 — Verification: confirm v0.1 works end to end

- [x] Sign out and open the dashboard → redirected to GitHub sign-in; sign in → the dashboard loads.
- [x] Register a product pointing at a repository → it appears in the list and survives a reload.
- [x] Edit then remove a product → both changes persist across a reload.
- [x] Point a product at a repo with a published release image → version reads `Ready` with its `ghcr.io` image reference.
- [x] Point a product at a repo with no image-publishing workflow → version reads `NoCi`.
- [x] Point a product at a repo that has releases but no built image → version reads `NeverBuilt`.

> `NoCi` / `NeverBuilt` confirmed via the E2E suite (18/18); the rest live-verified in-app. The version
> slice grew beyond the original three states (`UnreleasedBuild` / `BuildPending` / `BuildFailed`), and
> GHCR private-image auth (`read:packages` + Basic-auth token exchange) was proven against secrets-vault.
