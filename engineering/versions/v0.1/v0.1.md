# v0.1 — Sign in, register, find the ready version

*Last updated: 2026-06-12*

**Status:** 🚧 In Progress · **Started:** 2026-06-12 · **Completed:** —

> Iterations 1 (GitHub sign-in, HTTPS) + 2 (products CRUD) ✅ — verified live. Iteration 3 next.

Sign in with GitHub, register a product to its repo + published image, and confirm it has a ready version to deploy. Drydock ships pre-built images — building them is CI's job.

### Iteration 1 — Sign in ✅

**Goal:** Gate Drydock behind GitHub sign-in.

- [x] Require a GitHub sign-in to reach the dashboard (open / self-hosted; populate the allowlist to lock).
- [x] Keep the signed-in session across requests.

### Iteration 2 — Register products ✅

**Goal:** Manage the products Drydock can deploy.

- [x] CRUD a product — create, list, view, update, remove.
- [x] Select the repository that defines a product (its deploy source).

### Iteration 3 — Find its ready version

**Goal:** Confirm a product has a ready, published image to ship.

- [ ] Resolve the product's latest released version from its repository.
- [ ] Confirm that version's image is published and ready, or flag that none is.
