*Last updated: 2026-06-20*

## Cross-cutting risks (load-bearing)

- `/health` must stay **anonymous** under the fallback admin policy (re-apply `.AllowAnonymous()`).
- `SaveTokens=true` must survive the Identity adopt — `GhcrClient`/`GitHubClient` read `GetTokenAsync("access_token")`. Add an E2E assertion.
- Cookie name change (`.app.auth` default) invalidates live sessions → override to `.drydock.auth`.
- Integration-client extract needs a token-source abstraction (today hard-wired to `IHttpContextAccessor`).
- Every EXTRACT bumps the SDK beta version → adopt order matters (build leaf → publish → consume).
