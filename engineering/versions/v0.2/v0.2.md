# v0.2 — Full SDK adoption + extraction (everything but business logic)

*Last updated: 2026-06-20*

**Status:** 🚧 In Progress · **Started:** 2026-06-20 · **Completed:** —

> Extract-everything-while-small: anything reusable like lego — host, data contracts, web contracts,
> identity, the GitHub/GHCR integration clients, the test harness — is extracted to / adopted from the
> wow-two SDK, so future versions never pay the extraction tax. The ONLY things that stay are *how we
> use it*: the domain aggregates' meaning, the Products/Servers CQRS use-cases, the version-resolution
> **policy**, store predicates, and settings **values**. Detailed per-layer map: `v0.2-analysis.md`.

## Decisions (confirmed 2026-06-20)

- **D1 — STAYS = business logic only.** The 5 domain aggregates + their meaning, all `Products`/`Servers` CQRS verticals + DTOs, the version-resolution handler (deployability policy: `Ready`/`UnreleasedBuild`/…), store `Exists*` predicates, `ProductValidation`'s GitHub probe, settings VALUES. Everything else extracts or adopts.
- **D2 — Integration clients EXTRACT.** GitHub REST client → SDK `Integrations.GitHub`, GHCR client → `Integrations.Ghcr`. Token source parameterized (drop the hard `IHttpContextAccessor` coupling). drydock keeps the version-resolution policy that interprets them.
- **D3 — snake_case everywhere** — align to Postgres/platform. DB enum values snake (Iter 1 ✅); JSON wire flips to snake too (net-new SDK `Foundation.Serialization` preset to extract in Iter 3/4); frontend types flip in Iter 8. v0.1 migrations rewritable (data throwaway).
- **D4 — snake_case enum stored VALUES: migrate now** (`002-snake-enums`) while data is throwaway (only the `test` product exists). Else keep PascalCase.
- **D5 — Clock: adopt `TimeProvider`** (DateTimeOffset), NOT the SDK NodaTime `IClock`. 2 call-sites.
- **D6 — Timebox reality:** extract-everything exceeds the 1-week rule. Treat v0.2 as one multi-week *adoption cycle*, OR split into v0.2 (adopt + core web/data extracts) + v0.3 (integrations + identity leaves + testing). **Your call.**
- **D7 — SDK lane:** every EXTRACT is two-sided — build the leaf in `WoW2.Sdk.Backend.Beta`, bump its beta version, then adopt in drydock. Self-coordinated (you own both repos).
