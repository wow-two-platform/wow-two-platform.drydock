# v0.2 — Full SDK adoption + extraction (everything but business logic)

*Last updated: 2026-06-22*

**Status:** 🚧 In Progress · **Started:** 2026-06-20 · **Completed:** —

> Extract everything reusable (host · data · web · identity · integration clients · test harness) to the
> wow-two SDK; keep only *how we use it* — domain meaning, Products/Servers CQRS, version-resolution policy,
> store predicates, settings values. Per-layer map: `v0.2-analysis.md`.

## Decisions

- **D1** — STAYS = business logic only; everything else extracts or adopts.
- **D2** — GitHub + GHCR clients extract (`Integrations.*`), token source parameterized.
- **D3** — snake_case is **DB-only** (enum stored values + column names); the JSON wire stays **camelCase** (SDK `JsonOptionsPresets.Default`) — frontend types need **no** flip.
- **D4** — migrate enum stored values to snake now (`002-snake-enums`).
- **D5** — clock → SDK `TimeProvider`, not NodaTime.
- **D6** — extract-all exceeds the 1-week box → v0.2 runs as one multi-week adoption cycle.
- **D7** — every extract is two-sided: build SDK leaf → publish → adopt.
